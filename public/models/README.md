# Game + collectible assets

## Hand-tracking model (the 67)

`hand_landmarker.task` (~7.8 MB) powers the camera hand-tracking in the 67 game.
It plus the MediaPipe runtime under `public/mediapipe/` are generated by:

```
npm run setup:mediapipe
```

This copies the vision runtime from `@mediapipe/tasks-vision` and downloads the
hand-landmarker model from Google storage. The files are committed (self-hosted)
so the game has no runtime CDN dependency. Re-run the command if they go missing.

# 3D collectible models

Drop your collectible asset here as **`.glb`** (glTF binary):

```
public/models/collectible.glb
```

Then set the path in `src/lib/three-config.ts`:

```ts
export const COLLECTIBLE_MODEL_URL = "/models/collectible.glb";
```

The viewer (and the "Capture the moment" AR camera overlay) will load it
automatically, with a reveal + slow auto-rotate. If it appears too large/small,
tune `COLLECTIBLE_MODEL_SCALE` in the same file.

Notes:
- Use `.glb` (not `.obj`) — it bundles PBR materials + textures in one file.
- Keep it under ~5–10 MB; embed textures; Draco/meshopt compression is fine.
- If you only have `.obj`, export to `.glb` from Blender
  (File → Export → glTF Binary).
